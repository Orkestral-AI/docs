Um projeto é o lar de uma base de código dentro de um workspace. Ele aponta para um repositório ou pasta real e se torna o contexto em que sua equipe de IA trabalha. Quando você ativa um projeto, o orquestrador e seus especialistas leem aquele código, planejam com base nele e executam o trabalho como issues e epics.

Pense em um projeto como a resposta para "em qual código estamos trabalhando agora?". Você pode manter vários projetos em um workspace e alternar entre eles conforme passa de um produto ou serviço para outro.

<Info>
Projetos ficam dentro de um **workspace**. Se você ainda não configurou um workspace, comece por [Workspaces](/pt/workspaces) e depois volte aqui.
</Info>

## O que é um projeto

Cada projeto registra um conjunto pequeno e focado de fatos sobre uma base de código:

<CardGroup cols={2}>
  <Card title="Nome" icon="tag">
    Um rótulo curto que você reconhece, como `billing-api` ou `marketing-site`.
  </Card>
  <Card title="Tipo" icon="laptop">
    **Local** (uma pasta na sua máquina) ou **GitHub** (a URL de um repositório remoto).
  </Card>
  <Card title="Pasta ou URL do repositório" icon="folder-open">
    O caminho em disco para um projeto local, ou o remoto Git para um projeto GitHub.
  </Card>
  <Card title="Descrição" icon="align-left">
    Contexto opcional que ajuda sua equipe de IA a entender o objetivo da base de código.
  </Card>
</CardGroup>

Um projeto é intencionalmente leve. Ele não armazena seu código (isso fica no repositório), ele armazena o ponteiro para o seu código mais os metadados que seus agentes precisam para raciocinar sobre ele.

## Como um projeto agrupa seu trabalho

Uma vez que um projeto está ativo, tudo o que você cria flui para dentro dele. O orquestrador planeja com base no código do projeto ativo, e o trabalho que ele produz é acompanhado como issues e epics vinculados a esse projeto.

<CardGroup cols={3}>
  <Card title="Issues" icon="circle-dot">
    Uma única unidade de trabalho: um bug, uma pequena feature, uma mudança. Veja [Issues](/pt/issues).
  </Card>
  <Card title="Epics" icon="layer-group">
    Um objetivo maior dividido em várias issues que são entregues juntas. Veja [Epics](/pt/epics).
  </Card>
  <Card title="Chat" icon="comments">
    Conversas com seus agentes rodam com base no contexto do projeto ativo.
  </Card>
</CardGroup>

Esse agrupamento é o que mantém um workspace movimentado legível. Quando você troca de projeto, você troca a lente: o código sendo lido, o plano sendo feito e o trabalho sendo acompanhado, tudo se move junto com o projeto ativo.

<Note>
Apenas um projeto fica ativo por vez. O projeto ativo é mostrado em seu card como **Projeto ativo**, e ele determina o que o orquestrador e os especialistas enxergam.
</Note>

## Criar um projeto

<Steps>
  <Step title="Abra a página de Projetos">
    A partir do seu workspace ativo, abra **Projetos**. Você verá uma grade de projetos existentes, ou um estado vazio se este for o seu primeiro.
  </Step>
  <Step title="Inicie um novo projeto">
    Clique em **Novo projeto** no canto superior direito. O estado vazio também tem um botão **Crie seu primeiro projeto** que faz a mesma coisa.
  </Step>
  <Step title="Escolha o tipo">
    Escolha **Local** para apontar para uma pasta na sua máquina, ou **GitHub** para apontar para um repositório remoto.

    <Tabs>
      <Tab title="Local">
        Use isto para código que já vive no seu computador. Você escolherá uma pasta na próxima etapa.
      </Tab>
      <Tab title="GitHub">
        Use isto para um repositório hospedado no GitHub. Você colará a URL do repositório em vez de uma pasta.
      </Tab>
    </Tabs>
  </Step>
  <Step title="Nomeie o projeto">
    Digite um nome claro. Se você escolher uma pasta primeiro e deixar o nome vazio, o Orkestral preenche o nome a partir do último segmento do caminho da pasta automaticamente.
  </Step>
  <Step title="Aponte para o código">
    <Tabs>
      <Tab title="Local">
        Clique em **Escolher** para abrir o seletor de pastas nativo, ou cole o caminho diretamente no campo. O caminho é mostrado em monospace para você confirmar que está correto.
      </Tab>
      <Tab title="GitHub">
        Cole a URL do repositório (por exemplo, um remoto `https://` ou `git@`) no campo de URL do repositório.
      </Tab>
    </Tabs>
  </Step>
  <Step title="Adicione uma descrição (opcional)">
    Escreva uma frase ou duas sobre o que a base de código faz. Sua equipe de IA usa isso como contexto, então uma boa descrição se traduz em planos melhores.
  </Step>
  <Step title="Crie">
    Clique no botão de criar. Enquanto salva, você vê um spinner e **Criando**. Quando termina, o diálogo fecha e o novo projeto aparece na grade.
  </Step>
</Steps>

<Tip>
O campo de nome é obrigatório. O botão de criar permanece desabilitado até você digitar um nome, então, se parecer travado, verifique se o nome não está vazio.
</Tip>

## Tornar um projeto ativo

Um projeto só direciona sua equipe de IA quando é o projeto ativo.

<Steps>
  <Step title="Encontre o card do projeto">
    Na página **Projetos**, localize o card do projeto em que você quer trabalhar.
  </Step>
  <Step title="Torne-o ativo">
    Clique em **Tornar ativo** na parte de baixo do card. O card é destacado e o rótulo muda para **Projeto ativo**.
  </Step>
  <Step title="Comece a trabalhar">
    Crie issues, planeje um epic, ou abra um chat. Tudo isso agora roda com base no código deste projeto.
  </Step>
</Steps>

<Warning>
Trocar o projeto ativo muda o contexto para novos trabalhos. Issues e epics que você já criou permanecem vinculados ao projeto sob o qual foram feitos, então trocar não move o trabalho existente.
</Warning>

## Leia o card de relance

Cada card de projeto destaca os fatos principais sem precisar abrir nada:

<AccordionGroup>
  <Accordion title="Nome e pasta" icon="folder-tree">
    O topo do card mostra o nome do projeto. Para projetos locais, o caminho da pasta aparece logo abaixo em monospace para você confirmar para qual checkout ele aponta.
  </Accordion>
  <Accordion title="Descrição" icon="align-left">
    Se você adicionou uma descrição, ela aparece sob o nome, reduzida a algumas linhas. Este é o lembrete mais rápido de para que serve a base de código.
  </Accordion>
  <Accordion title="Tipo e remoto" icon="code-branch">
    Uma pequena linha mostra se o projeto é **Local** ou **GitHub**, ao lado do remoto Git quando há um definido. Isso indica de relance onde o código vive.
  </Accordion>
  <Accordion title="Estado ativo" icon="circle-check">
    O card do projeto ativo é destacado, e sua ação exibe **Projeto ativo** em vez de **Tornar ativo**. Use isso para confirmar em qual projeto sua equipe está trabalhando no momento.
  </Accordion>
</AccordionGroup>

## Acompanhe o progresso

Você acompanha um projeto observando o trabalho que vive dentro dele avançar.

<Steps>
  <Step title="Torne o projeto ativo">
    Defina o projeto como ativo para que novos trabalhos e o chat sejam delimitados a ele.
  </Step>
  <Step title="Abra Issues">
    Vá para [Issues](/pt/issues) para ver cada unidade de trabalho do projeto e seu status atual, do backlog ao pronto.
  </Step>
  <Step title="Agrupe trabalhos grandes em epics">
    Para objetivos maiores, use [Epics](/pt/epics) para juntar issues relacionadas e acompanhar o esforço inteiro como um único fio.
  </Step>
  <Step title="Deixe a equipe executar">
    O orquestrador planeja, os especialistas refinam, e o Forge local aplica as mudanças de código. Observe as issues percorrerem seus status conforme o trabalho é entregue.
  </Step>
</Steps>

<Info>
O progresso é a soma das suas issues e epics, não um número separado que você mantém à mão. Mantenha o trabalho fluindo como issues, e o status do projeto acompanha.
</Info>

## Remover um projeto

Quando uma base de código não está mais em jogo, você pode remover seu projeto.

<Steps>
  <Step title="Passe o mouse sobre o card">
    Na página **Projetos**, passe o mouse sobre o card do projeto. Um ícone de lixeira aparece no canto inferior direito do card.
  </Step>
  <Step title="Clique no ícone de lixeira">
    Clique nele para iniciar a remoção. O Orkestral pede sua confirmação, nomeando o projeto para você não remover o errado.
  </Step>
  <Step title="Confirme">
    Confirme o aviso. O projeto é removido do workspace. Se ele era o projeto ativo, a seleção ativa é limpa.
  </Step>
</Steps>

<Warning>
Remover um projeto o remove do seu workspace. Isso não apaga seu código real em disco nem seu repositório remoto; eles permanecem exatamente onde estão.
</Warning>
